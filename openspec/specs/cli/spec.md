# Cli Specification

> Generated by spec-gen v1.0.0 on 2026-03-25 21:54

## Purpose

Provides interactive prompts for user input during the spec-gen CLI workflow.

## Requirements

### Requirement: Promptoverwrite

The system SHALL prompts user to choose how to handle existing spec files.

#### Scenario: FileExists
- **GIVEN** A spec file already exists at the specified path
- **WHEN** promptOverwrite is called
- **THEN** User is presented with options to backup, merge, skip, or skip all

### Requirement: Confirmoverwriteall

The system SHALL confirms with user before overwriting multiple files.

#### Scenario: MultipleFiles
- **GIVEN** Multiple spec files exist
- **WHEN** confirmOverwriteAll is called
- **THEN** User is asked to confirm before proceeding with overwrite

### Requirement: Confirmlonganalysis

The system SHALL confirms with user before starting a potentially long analysis.

#### Scenario: LargeCodebase
- **GIVEN** A large number of files need to be analyzed
- **WHEN** confirmLongAnalysis is called
- **THEN** User is warned about potential duration and asked to confirm

### Requirement: Confirmgeneration

The system SHALL confirms with user before making api calls for generation.

#### Scenario: ApiCallConfirmation
- **GIVEN** Ready to make API calls for generation
- **WHEN** confirmGeneration is called
- **THEN** User is shown model and token estimates and asked to confirm

### Requirement: Promptapikey

The system SHALL prompts user to enter api key when not found in environment.

#### Scenario: MissingApiKey
- **GIVEN** No API key found in environment variables
- **WHEN** promptApiKey is called
- **THEN** User is guided to get an API key and prompted to enter it

### Requirement: Selectprovider

The system SHALL allows user to select llm provider when multiple are available.

#### Scenario: MultipleProviders
- **GIVEN** Multiple API keys are available
- **WHEN** selectProvider is called
- **THEN** User is presented with options to choose between providers

### Requirement: Selectsamplecount

The system SHALL allows user to select number of files to verify.

#### Scenario: VerificationSampleSelection
- **GIVEN** Need to select number of files for verification
- **WHEN** selectSampleCount is called
- **THEN** User is presented with predefined options or can enter custom number

### Requirement: Promptcontinueaftererror

The system SHALL prompts user to continue after a partial failure.

#### Scenario: PartialFailure
- **GIVEN** An error occurred during a stage
- **WHEN** promptContinueAfterError is called
- **THEN** User is shown error details and asked if they want to continue

### Requirement: Selectdomains

The system SHALL allows user to select domains to generate specs for.

#### Scenario: DomainSelection
- **GIVEN** Multiple domains are available
- **WHEN** selectDomains is called
- **THEN** User can choose to generate for all domains or select specific ones

### Requirement: Isinteractive

The system SHALL checks if the application is running in interactive mode.

#### Scenario: InteractiveModeCheck
- **GIVEN** Need to determine if running in interactive mode
- **WHEN** isInteractive is called
- **THEN** Returns true if running in interactive mode, false otherwise

### Requirement: Setinteractivemode

The system SHALL enables or disables interactive mode.

#### Scenario: SetInteractiveMode
- **GIVEN** Need to change interactive mode setting
- **WHEN** setInteractiveMode is called
- **THEN** Interactive mode is enabled or disabled based on input

### Requirement: GeneratePreflightExemptsCursorAgentFromApiKeys

The `spec-gen generate` workflow (CLI command and programmatic entrypoints that perform the same API-key preflight) SHALL treat `cursor-agent` as a provider that does not require `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GEMINI_API_KEY`, or `OPENAI_COMPAT_API_KEY` to be set for pre-flight validation.

Authentication for Cursor SHALL remain the responsibility of the Cursor CLI and Cursor’s documented auth mechanisms (not spec-gen cloud keys).

#### Scenario: CliGenerateWithCursorAgentOnly

- **GIVEN** `.spec-gen/config.json` sets `generation.provider` to `cursor-agent`
- **AND** none of `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GEMINI_API_KEY`, or `OPENAI_COMPAT_API_KEY` are set
- **WHEN** the user runs `spec-gen generate` with otherwise valid options
- **THEN** pre-flight does not fail solely due to missing those environment variables

#### Scenario: ProgrammaticGenerateWithCursorAgentOnly

- **GIVEN** programmatic generation is invoked with provider `cursor-agent`
- **AND** none of `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GEMINI_API_KEY`, or `OPENAI_COMPAT_API_KEY` are set
- **WHEN** generation proceeds past provider resolution
- **THEN** pre-flight does not throw solely due to missing those environment variables
